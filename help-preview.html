Usage: merge_documents.py [-h] [-s FILEPATH or -] [--confirm-exit | --no-confirm-exit] [-q | --quiet | --no-quiet] [-whatif] [-l IDENTIFIER] [-c PATH_TO_CONFIG]
                          [--recursion-limit RECURSION_LIMIT] [-p PAGE_SIZE] [-m MARGIN] [--fp | --no-fp | --force-image-page-fallback-size | --no-force-image-page-fallback-size]
                          [--afs | --no-afs | --alphabetic-file-sorting | --no-alphabetic-file-sorting] [--libreoffice-path [LIBREOFFICE_PATH ...]]
                          [-od OUTPUT_DIRECTORY | -o OUTPUT_FILEPATH]
                          [files ...]

Merges given PDFs, images and office document formats into a single PDF file.

The program accepts both file and folders (which are recursively scanned for supported files.)

Most of the parameters can be loaded from a configuration file.
A configuration file with default values is automatically created if it does not exists.
Its path is config.toml. It will always be loaded when the program runs (unless a different file is specified via --config)

Flag options have their negative variant to override parameters loaded from the configuration file.
(With the default configuration file they are not needed)

Default values shown in help are not loaded from any configuration file.

Positional Arguments:
  files
        Directories and files to be processed.
        Directories will be searched recursively looking for images, pdfs and office document formats.
        Relative paths are based in the current working directory.
        (default: None)

Options:
  -h, -?, --help, /?
        Show this help message and exit. Running the app without any files does the same.
  -s, --save-config FILEPATH or -
        If a path is provided, all input parameters are saved as a TOML configuration file, under the given path.
        Input parameters are a union of commandline parameters as well as the loaded configuration file's parameters.
        To send the TOML text to standard output, specify '-' as the destination.
        After saving the condfiguration the program exits.
        (default: None)
  --confirm-exit, --no-confirm-exit
        If True, will not exit the program until user confirms.
        (default: False)
  -q, --quiet, --no-quiet
        If True, prints no messages to the console. Overrides --confirm-exit.
        (default: False)
  -whatif, --what-if, --whatif
        If present, runs the program, but outputs no files. Overrides --quiet.
        (default: False)
  -l, --language IDENTIFIER
        Which language file to load. Language files should follow the 'language.{IDENTIFIER}.json' pattern where 'IDENTIFIER' is an identifier for the language.
        This argument accepts this identifier.
        If the identifier is an empty string, first available file is loaded.
        If the matching file is not found, default language will be used.
        (default: "")

Input Parameters:
  All parameters in this group can be stored in a configuration file. When running the application first the configuration file is read, then it is updated with commandline parameters.

  -c, --config PATH_TO_CONFIG
        Custom path to a configuration file. Will be used in place of the default configuration file.
        Some parameters cannot be saved (e.g. files or output_file).
        (default: None)
  --recursion-limit RECURSION_LIMIT
        How deep to search for supported files in a directory.
        (default: 5)
  -p, --image-page-fallback-size PAGE_SIZE
        When adding images, the page size is set to first PDF's first page size.
        If there are no PDF files provided, the fallback value is used.
        Format: "(width)(unit) x (height)(unit)". Units have to match.
        You can use points (pt), millimetres (m), centimetres (cm) and inches (inch) as units. Default unit is point.
        Can also specify common paper sizes, like 'A4'.
        (default: A4)
  -m, --margin MARGIN
        Margin to be used when adding images.
        Format: "(width)(unit) x (height)(unit)". Units have to match.
        You can use points (pt), millimetres (m), centimetres (cm) and inches (inch) as units. Default unit is point.
        (default: 0mm x 0mm)
  --fp, --no-fp, --force-image-page-fallback-size, --no-force-image-page-fallback-size
        If True, the specified fallback size will always be used for image pages, regardless of other PDF files.
        (default: False)
  --afs, --no-afs, --alphabetic-file-sorting, --no-alphabetic-file-sorting
        When true the order in which the paths are specified is ignored - all paths are sorted alphabetically as if they were in one directory.
        Useful when using the program by dragging the files onto the executable.
        (default: False)
  --libreoffice-path [LIBREOFFICE_PATH ...]
        Path to Libre Office executable.
        First available path will be used.
        If no path is valid conversion of document formats is disabled.
        Paths may contain '~' and environmental variables (surrounded with '%' or prepended with '$').
        You can use both forward and backward slashes. Backward slashes need to be doubled.
        Can be relative (to the current working directory).
        (default: ['%PROGRAMFILES%\\LibreOffice\\program\\soffice.exe', '%PROGRAMFILES(X86)%\\LibreOffice\\program\\soffice.exe'])

Output:
  You can specify either output filepath or output directory. When the directory path is provided the output filename will be generated based on the current date.

  -od, --output-directory OUTPUT_DIRECTORY
        Path of the directory where the output file will be placed. Filename will be generated based on time and language.
        This path will always be treated as a folder path, even if you provide an extension.
        (default: .)
  -o, -of, --output-file OUTPUT_FILEPATH
        Path of the output file. Relative to the current working directory.
        This path will always be treated as a file path, even if you do not provide an extension.
        Extension will be changed to ".pdf/.png" as needed.
        (default: None)

To use the program by drag&dropping files and folders onto it, enable the config_dragdrop.toml which should be provided.

To enable change it name to config.toml, or add --config config_dragdrop.toml as an arguments to the shortcut or script you want to drop file onto.
